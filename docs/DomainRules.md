# 311통신대대 카풀방 도메인 규칙

## 핵심 개념
- 본 서비스는 `당일 출근 가능 여부`를 먼저 판정한 뒤, 차량 보유/홀짝 규칙/좌석 상태를 순차 적용한다.
- 차량 운행 가능 여부는 `출근 가능 AND 차량 보유 AND 홀짝 일치`로 판단한다.
- 탑승 필요 여부는 `출근 가능 AND (차량 미보유 OR 운행 불가)`로 판단한다.

## 상태 분류

### 사용자 당일 근무 상태
- `ON_DUTY`: 출근 가능
- `OFF_DUTY`: 휴가/휴무/청원휴가/근취 등으로 출근 불가

### 사용자 이동 상태
- `DRIVER_AVAILABLE`: 운행 가능 차량 보유, 당일 운행 가능
- `RIDER_REQUIRED`: 당일 탑승 필요
- `NO_ACTION`: 당일 출근 불가(OFF_DUTY)

### 탑승 신청 상태
- `WAITING`: 좌석 부족으로 대기
- `APPROVED`: 탑승 확정
- `REJECTED`: 운전자 거절
- `CANCELLED`: 신청자 취소

## 판정 우선순위
1. 비출근일 확인(`OFF_DUTY`면 즉시 `NO_ACTION`)
2. 차량 보유 여부 확인
3. 차량 번호 끝자리와 당일 홀짝 규칙 일치 여부 확인
4. 운행 등록(`ride_offer`) 여부 확인
5. 좌석 대비 신청 수로 확정/대기 상태 배정

## 2부제 규칙
- 기본 규칙은 날짜(day-of-month) 홀짝으로 계산한다.
  - 날짜 홀수 -> 홀수 차량 운행 가능
  - 날짜 짝수 -> 짝수 차량 운행 가능
- 운영자가 특정 날짜에 예외 규칙을 지정할 수 있다.
  - `ALL_ALLOWED`: 모든 차량 운행 가능
  - `ODD_ONLY`: 홀수만 가능
  - `EVEN_ONLY`: 짝수만 가능
  - `NONE_ALLOWED`: 운행 금지

## 웨이팅 규칙
- 신청 시점 기준으로 `queue_order`를 부여한다.
- `APPROVED` 수가 `seat_capacity`보다 적으면 즉시 승인 가능하다.
- 승인/거절/취소 발생 시 대기열에서 가장 빠른 `WAITING` 신청자를 자동 승격한다.
- 동일 운행건(`ride_offer`)에 대해 신청자는 1건만 활성 상태(`WAITING`/`APPROVED`)를 가질 수 있다.

## 예시 시나리오
- 사용자 A: 차량 보유, 당일 비출근일 아님, 홀짝 일치 -> `DRIVER_AVAILABLE`
- 사용자 B: 차량 보유, 당일 비출근일 아님, 홀짝 불일치 -> `RIDER_REQUIRED`
- 사용자 C: 차량 없음, 당일 비출근일 아님 -> `RIDER_REQUIRED`
- 사용자 D: 차량 보유, 당일 휴가 -> `NO_ACTION`
